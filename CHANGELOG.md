# Changelog

本项目的所有重要更新都会记录在这里。

格式参考 Keep a Changelog，并遵循语义化版本管理思路。

## [Unreleased]

## [v1.6.0] - 2026-07-11

### Added

- 新增 `/bk版本`、`/bk检查更新` 和 `/bk更新插件` 命令
- 优先通过 GitHub Releases 检查更新；没有 Release 时读取仓库默认分支的 `metadata.yaml`
- 新增安全更新安装器：版本和插件名校验、必要文件校验、Zip Slip 防护、下载大小限制、旧文件备份与失败回滚
- 新增 `updates.self_update_enabled` 与 `updates.max_download_mb` 配置

### Security

- 更新源固定为 `jiuhunwl/astrbot_plugin_bktools`，更新命令沿用插件管理员权限控制
- 单群高级配置不能开启全局已关闭的自更新功能
- GitHub 下载和重定向继续应用公网 URL、DNS 和重定向目标安全校验

### Fixed

- 4xx 响应不再累计接口熔断失败次数，只对网络错误、超时和 5xx 计入熔断

## [v1.5.1] - 2026-07-11

### Fixed

- 修复 AstrBot v4.25.5 配置对象与 `copy.deepcopy()` 不兼容导致事件处理入口报 `'NoneType' object is not callable` 的问题
- 配置初始化、群策略解析和高级覆盖合并改为递归生成普通 Python 容器，避免触发框架对象的 `__setstate__`/动态属性协议
- 新增 AstrBotConfig 风格 `dict` 子类回归测试

## [v1.5.0] - 2026-07-11

### Added

- 新增独立运行时模块，统一管理任务状态、发送幂等键、临时文件、接口健康和熔断状态
- 解析接口支持网络错误、超时和 5xx 有限重试，并按接口提供失败计数、熔断和自动恢复
- 超长 JSON 结果改为单个 `bktools-result.json` 文件回退，禁止拆成逐作品消息
- 新增接口响应、JSON 文件和运行时缓存容量上限

### Changed

- 解析接口与安全短链展开复用统一 `aiohttp.ClientSession`，插件卸载时主动关闭
- `/bk状态` 增加任务状态、HTTP 会话和熔断数量，`/bk诊断` 增加接口健康与脱敏错误
- 单群高级配置不能覆盖全局 `runtime_limits` 资源安全限制
- 打包文件新增 `bktools_runtime.py`，版本统一更新为 v1.5.0

### Fixed

- 同一会话同一解析结果增加发送幂等保护，降低命令与自动触发并发造成的重复输出
- 停止、失败和卸载流程统一回收任务登记的临时 JSON 文件

## [v1.4.0] - 2026-07-10

### Security

- **修复任意 URL 请求/SSRF 风险**：自动触发不再请求非支持平台 URL；媒体下载、文件大小检测和短链接重定向均校验协议、凭据、端口、DNS 结果及每次重定向目标
- **严格域名匹配**：使用主机名边界匹配，拒绝 `douyin.com.evil.example` 等伪装域名
- **保护安全配置**：单群 `override_json` 不能覆盖 `group_control` 和 `security`，避免群级配置降低全局安全策略
- **新增管理命令权限**：停止解析、清理缓存、状态和诊断命令默认仅允许群主、群管理员或全局授权用户执行

### Added

- **真正取消解析任务**：按群/私聊登记活动任务，停止命令使用任务取消中断 HTTP、下载和等待中的解析
- **单会话并发限制**：可配置每个群或私聊最多同时执行的解析任务数
- **运行时缓存回收**：定期清理去重、点歌和完成后的解析锁缓存，卸载插件时释放任务、缓存和临时文件
- **新增 `/bk状态`**：查看当前会话有效配置、启用功能、活动任务和缓存数量
- **新增 `/bk诊断`**：检查接口地址、ffmpeg、pydub、Alist 配置和 URL 安全防护
- **新增自动化回归测试**：覆盖 SSRF、伪装域名、权限控制、群白名单、任务取消和缓存回收

### Fixed

- **清理范围不完整**：缓存清理现同时处理音频、视频和语音转换临时文件
- **解析锁残留**：任务结束后直接移除锁键，不再长期保留 `False` 状态条目

## [v1.3.0] - 2026-07-10

### Added

- **新增群聊启用范围控制**：默认对所有群启用，也可切换为“仅单群配置列表中的群启用”；未设置时保持原有全开启行为
- **新增单群功能开关**：可分别控制短视频作品、抖音主页、网易云搜歌和音乐链接解析，关闭后自动触发与手动命令同时停用
- **新增单群行为覆盖**：支持独立覆盖自动解析、合并转发、解析提示、音乐语音、大视频直链和批量 JSON 输出
- **新增高级群配置 JSON**：可为某群覆盖任意全局配置，例如图片展示数、视频阈值、重试参数或接口地址
- **新增平台限定匹配**：相同群聊 ID 可按 `aiocqhttp`、`telegram`、`discord` 等平台分别配置

### Changed

- **任务状态按群隔离**：停止解析仅中断当前群任务，不再影响其他群或私聊会话
- **去重与解析锁按群隔离**：同一链接可在不同群正常解析，同时继续阻止单群内重复触发
- **群配置使用任务级上下文**：并发群聊各自使用独立有效配置，避免临时覆盖全局配置产生串群

## [v1.2.9] - 2026-07-10

### Changed

- **重新设计插件设置页**：常用配置前置，设置分组增加图标、清晰标题和用途说明，减少字段堆叠感
- **高级配置按需显示**：短视频、网易云、其他音乐平台和 HTTP 字段映射默认收起，通过“显示高级设置”开关展开
- **关联配置动态显示**：重试参数、语音限制、Alist 凭据、视频阈值、批量 JSON 阈值和调试长度仅在对应功能启用时显示
- **优化输入控件**：数值范围改用滑块与数字输入组合，请求方式增加可读下拉标签，JSON 参数启用代码编辑器
- **统一配置文案**：补齐字段名称、提示、建议范围和风险说明，移除未使用且会产生空白分组的缓存配置项

### Fixed

- **修复“附带原始分享链接”设置未生效**：开启后会在短视频文字信息末尾真实附加原始链接
- **修复关闭 Video 组件后的发送异常**：`pack_send_video=false` 时正确使用视频直链，不再因未初始化变量中断发送，也不会在元信息与资源节点中重复展示同一链接

## [v1.2.8] - 2026-07-10

### Fixed

- **修复停止解析偶发失效**：使用解析任务取消代次替代会被新任务重置的全局布尔标志，停止指令可稳定截断所有已启动的短视频与主页解析输出
- **修复合并转发与单条消息同时发送**：合并转发失败后不再逐作品降级发送，避免平台已接收打包消息但客户端抛错时产生混合输出
- **修复主页作品打包节点嵌套**：已构造的 `Node` 不再被重复包装，提升群聊合并转发兼容性

### Added

- **新增合并转发重试**：支持配置 `pack_retry_count` 与 `pack_retry_delay_ms`；重试全部失败后仅发送接口返回的完整 JSON
- **恢复插件打包脚本**：可使用 `python package_plugin.py` 生成 AstrBot WebUI 可直接上传的 zip

## [v1.2.4] - 2026-06-08

### Added

- **新增批量内容输出控制**：图集/实况数量超过阈值时可直接输出JSON响应

## [v1.2.3] - 2026-05-16

### Added

- **视频文件大小阈值控制**：增加文件大小阈值判断机制，当视频文件超过设定值时自动发送资源链接而非直接传输文件
  - 新增 `video_threshold` 配置组，包含以下参数：
    - `force_link_enabled`：全局强制链接发送总开关（优先级最高，启用时始终发送链接）
    - `threshold_enabled`：阈值触发链接发送功能开关（仅在全局开关关闭时生效）
    - `threshold_mb`：文件大小阈值（MB），默认 100MB
  - 通过 HEAD 请求检测远程视频文件大小，避免不必要的下载开销
  - 完善的错误处理：文件大小检测失败时自动回退到正常发送流程
